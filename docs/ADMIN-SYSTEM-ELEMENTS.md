# Ant Design Pro 类后台系统：知识点与必要实现元素总表

更新时间：2026-05-28

## 1. 文档用途
- 学习视角：你需要掌握哪些知识点。
- 实战视角：一个可用后台系统必须具备哪些元素。
- 复盘视角：每次迭代可以对照本表查漏补缺。

## 2. 系统级必要元素（最小可用）
- [ ] E-01 应用壳布局（Header/Sider/Content/Footer）
- [ ] E-02 路由系统（含 404）
- [ ] E-03 登录态管理
- [ ] E-04 菜单导航与高亮联动
- [ ] E-05 权限控制（至少路由级）
- [ ] E-06 统一请求层与错误处理
- [ ] E-07 至少一个完整业务 CRUD 页面
- [ ] E-08 基础工程规范（lint/build 可通过）

## 3. 知识点全景地图（按模块）

### 3.1 React 基础与进阶
- 组件设计与组合
- 状态管理与数据流
- 副作用模型（`useEffect`）
- 自定义 hooks 复用逻辑
- 性能优化（memo/useMemo/useCallback 认知）

### 3.2 路由与导航
- 静态路由配置
- 动态路由映射（后端菜单）
- 嵌套路由与布局路由
- 路由守卫与重定向
- 面包屑和菜单同步策略

### 3.3 权限体系
- RBAC 基础模型
- 路由级权限
- 菜单级权限
- 按钮级权限
- 异常页（403/404/500）

### 3.4 数据与状态
- React Query（query/mutation/invalidate）
- Zustand（全局 UI/会话状态）
- 服务端状态与客户端状态边界
- 请求并发、重试、取消、错误兜底

### 3.5 页面开发模板能力
- 列表页（搜索 + 表格 + 分页）
- 表单页（新建/编辑 + 校验 + 提交态）
- 详情页（数据展示 + 操作记录）
- 弹窗/抽屉表单（二级操作）
- 导入导出（可选）

### 3.6 系统管理模块
- 用户管理
- 角色管理
- 菜单管理
- 部门组织（可选）
- 字典与配置管理（可选）

### 3.7 体验与产品能力
- 主题切换
- 国际化
- 标签页导航
- 全局消息通知
- 空状态/骨架屏/加载态一致性

### 3.8 工程化与质量保障
- TypeScript 类型体系
- ESLint/格式化/提交规范
- 单元测试与关键流程测试
- CI（lint/build/test）
- 构建体积与性能优化

### 3.9 服务层与 BFF（Nitro）
- 本地 mock/BFF 接口
- 动态菜单和权限接口模拟
- 鉴权中间件模拟
- 缓存与存储基础认知
- mock 到真实后端的平滑替换

## 4. 实现清单（按“必须/建议/进阶”）

### 必须实现（MUST）
