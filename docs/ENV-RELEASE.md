# 环境与发布文档

更新时间：2026-05-28

## 1. 目标
- 统一开发、测试、生产环境配置方式。
- 固化可重复的发布与回滚流程。

## 2. 环境划分
- `dev`：本地开发与联调
- `test`：测试环境（提测、验收）
- `prod`：生产环境

## 3. 环境变量约定（初版）
| 变量名 | 说明 | 示例 |
| --- | --- | --- |
| `VITE_APP_ENV` | 当前环境标识 | `dev` |
| `VITE_API_BASE_URL` | API 基础地址 | `/api` |
| `VITE_ROUTE_SOURCE` | 路由来源模式 | `frontend` |
| `VITE_ENABLE_NITRO_MOCK` | 是否启用 Nitro mock | `true` |

说明：
- 前端只读取 `VITE_` 前缀变量。
- 所有环境变量都要在文档中登记，不允许“隐式变量”。

## 4. 本地开发流程
1. 安装依赖：`pnpm install`
2. 启动开发：`pnpm dev`
3. 质量检查：`pnpm lint`
4. 构建验证：`pnpm build`

## 5. 发布前检查清单
- [ ] 需求范围已完成并自测
- [ ] `pnpm lint` 通过
- [ ] `pnpm build` 通过
- [ ] 关键路径验证（登录、菜单、主要页面）
- [ ] 文档联动更新完成
- [ ] Git 提交记录清晰、可追溯

## 6. 发布流程（简化版）
1. 从功能分支合并到主线分支（按团队策略）。
2. 打标签（可选）：`vX.Y.Z`
3. 构建产物并部署到目标环境。
4. 发布后做冒烟测试（关键路径）。

## 7. 回滚流程（简化版）
1. 确认回滚目标版本（tag/commit）。
2. 回滚部署到上一个稳定版本。
3. 验证关键路径恢复正常。
4. 补充回滚原因与后续修复计划。

## 8. Nitro 相关约定
- 开发阶段可启用 Nitro mock。
- 切换到真实后端时，只切换 `VITE_API_BASE_URL` 与 mock 开关。
- 页面层不得依赖 mock 专属字段。
