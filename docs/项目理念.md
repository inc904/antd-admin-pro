# 我的 React 企业级后台学习项目说明

## 为什么会有这个项目

这个项目的目标不是做一个后台系统。

而是通过开发一个真实可用的后台系统，系统化学习企业级前端开发。

我希望通过这个项目，把零散的知识点串联起来，建立完整的工程认知。

最终能够独立设计、开发、维护一个企业级 React 项目。

---

# 我希望 Codex 扮演什么角色

Codex 不是我的外包程序员。

Codex 是我的开发教练。

它的价值不在于帮我写代码。

而在于帮助我理解代码。

---

## 理想的协作方式

当我遇到问题时：

Codex 应该：

* 解释知识点
* 分析原因
* 给出思路
* 提供示例
* 指出修改位置

然后让我自己完成修改。

因为：

自己修改一次，

比看十次答案更容易学会。

---

## 为什么不希望 Codex 默认改代码

如果 AI 直接改代码：

问题会立刻解决。

但能力不会真正提升。

长期来看：

我会越来越依赖 AI。

而不是越来越理解项目。

所以：

默认情况下，

源码由我修改。

知识由 Codex 讲解。

---

# 文档的重要性

企业项目不仅是代码。

文档同样重要。

我希望在这个项目里养成：

* 写任务记录
* 写知识总结
* 写设计决策
* 写 API 契约
* 写开发日志

的习惯。

让项目成长过程可以被回顾和复盘。

---

# Git 的意义

提交记录不仅是备份。

也是成长轨迹。

每一次提交都应该代表：

一个清晰的阶段成果。

我希望通过这个项目学习：

* 如何拆分任务
* 如何规划里程碑
* 如何编写 Commit Message
* 如何维护开发日志

---

# 项目成功的标准

这个项目是否成功，

不取决于后台系统做得多复杂。

而取决于：

当项目结束时，

我是否已经具备独立完成企业级 React 项目的能力。

---

# 项目原则

源码我来写。

原理你来教。

文档一起维护。

提交谨慎记录。

优先培养能力。

而不是优先完成任务。
